Core Architecture:

1. BEHAVIORS (State Machine)
   - core.ts: Generic behavior machine implementation
   - expense.behavior.ts: Expense workflow definition

2. DATABASE
   - database.ts: SQLite connection & schema
   - migrate.ts: Create tables
   - seed.ts: Insert sample data

3. SERVICES (Business Logic)
   - expense.service.ts: Expense operations
   - category.service.ts: Category CRUD
   - user.service.ts: User management

4. ROUTES (API Layer)
   - expenses.ts: RESTful expense endpoints
   - categories.ts: Category endpoints
   - users.ts: User endpoints

5. FRONTEND
   - index.html: Complete web interface
   - Pure HTML/CSS/JavaScript (no framework)

Technology Stack:

- Runtime: Bun (v1.0+)
- Web Framework: Hono
- Database: SQLite (built into Bun)
- Language: TypeScript
- Testing: Bun Test
- Frontend: Vanilla JavaScript

API Endpoints:

Expenses:
  GET    /api/expenses                  # List all
  POST   /api/expenses                  # Create
  GET    /api/expenses/:id              # Get by ID
  PATCH  /api/expenses/:id              # Update
  DELETE /api/expenses/:id              # Delete
  POST   /api/expenses/:id/submit       # Submit
  POST   /api/expenses/:id/approve      # Approve
  POST   /api/expenses/:id/reject       # Reject
  POST   /api/expenses/:id/pay          # Mark paid
  POST   /api/expenses/:id/archive      # Archive
  GET    /api/expenses/:id/history      # Get history
  GET    /api/expenses/stats/summary    # Statistics

Categories:
  GET    /api/categories                # List all
  POST   /api/categories                # Create
  GET    /api/categories/:id            # Get by ID
  PATCH  /api/categories/:id            # Update
  DELETE /api/categories/:id            # Delete

Users:
  GET    /api/users                     # List all
  POST   /api/users                     # Create
  GET    /api/users/:id                 # Get by ID

Database Schema:

Tables:
  - users              (User accounts & roles)
  - categories         (Expense categories)
  - expenses           (Main expense records)
  - expense_history    (State transition audit log)
  - receipts           (File attachments - planned)

State Flow:

  draft → pending_approval → approved → paid → archived
            ↓                    ↓
          rejected ←─────────────┘

States:
  - draft              Initial creation
  - pending_approval   Awaiting review
  - approved           Approved by manager
  - rejected           Rejected by manager
  - paid               Payment processed
  - archived           Archived for records

Commands:

Development:
  bun install          # Install dependencies
  bun run dev          # Start dev server (watch mode)
  bun run start        # Start production server

Database:
  bun run db:migrate   # Initialize database
  bun run db:seed      # Add sample data

Testing:
  bun test             # Run test suite

Environment Variables:

PORT=3000                          # Server port
DATABASE_PATH=./expenses.db        # SQLite database path
NODE_ENV=development               # Environment (development/production)
